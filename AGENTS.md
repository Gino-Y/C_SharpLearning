# CSharpLearning 结构化笔记规则

> 本文件是本仓库笔记整理工作的**唯一真源**（人与 AI 共用）。从课例代码归档到 Obsidian 可导航知识库。宗旨：**信息密度** + **层级结构**；文件图深度 **≤ 3 层**。
>
> 导航原则：**本级详述 + 子级简报** — 无论处于哪一层，只阅读本级完整内容，以及下一层各节点的一句话摘要；子层细节留给点入后再读。
>
> 词表摘要与 Cursor 短规则见 [`Notes/README.md`](./Notes/README.md)、[`.cursor/rules/notes-tags.mdc`](./.cursor/rules/notes-tags.mdc)；**细则以本文件为准**。

---

## 一、文件关系规则

### R0 · 本级详述 + 子级简报

每一层文件只承载两类信息，**不预载、不展开子层正文**：

| 部分 | 含义 |
|------|------|
| **本级详述** | 当前层自身应完整表达的内容 |
| **子级简报** | 每个直接子节点一行：可点击标题 + 一句话描述 |

| 层级 | 本级详述 | 子级简报 |
|------|----------|----------|
| **L1**（README、枢纽索引） | 库规则摘要、组织方式、快速入口 | 各 L2 模块 / 各枢纽一行摘要 + 链接 |
| **L2**（模块速查卡） | 本模块知识脉络、口诀、依赖说明 | 各 L3 篇目一行摘要 + 链接 |
| **L3**（篇目） | 完整知识点（最底层，无子级简报） | — |

**禁止在本层出现：**

- 父层的详细推导或背景叙事
- 子层篇目的正文、代码、长列表复制
- 把多层内容堆在同一文件里（「目录页 + 全文合集」）

**子级简报标准格式：**

```markdown
- [**篇目标题**](./路径/篇目.md)：一句话说明本篇讲什么、解决什么问题
```

L3 是最底层：打开即全文，不再嵌套下级摘要。

---

### R1 · 三层文件图

| 层级 | 名称 | 职责 | 命名示例 |
|------|------|------|----------|
| **L1** | 库级 | 整门课入口、跨模块导航 | `Notes/README.md`、`Notes/枢纽/00_枢纽索引.md` |
| **L2** | 模块级 | 按知识域划分的课程簇 | `01-基础语法.md`、`02-面向对象.md`（速查卡） |
| **L3** | 篇目级 | 一个知识点 / 一道练习 / 一个实战 | `索引器.md`、`委托-练习1.md` |

- **禁止 L4 目录**（如 `Class02-面向对象/集合/List/`）。
- 粒度到「知识点」为止；再细则拆成新 L3 篇目，而非加深文件夹。
- **不强制**重命名现有文件夹为 `编号_主题/`；保持 `Class01-基础语法/` 等现状路径。

---

### R2 · 存储分工

| 目录 / 位置 | 角色 | 是否计入三层学习图 |
|-------------|------|-------------------|
| `Class01/`、`Class02/`、`TLS_HX/`、`Example/` | 课例源码（`.cs`） | 否（溯源） |
| `Notes/` | 结构化笔记主图 | 是（L1–L3 均在此） |
| `Notes/Attachments/` | 音频/配图资源 | 否（资源层） |
| `Notes/枢纽/` | 跨 L2 概念汇聚 | 是（L1） |
| `Notes/检查表/` | 动手时的步骤清单 | 是（L1 附属） |

- 学习导航只走 `Notes/`；课例代码仅作「对应课件」溯源，**主链不经过源码目录**。
- L3 用 frontmatter `对应课件:` 指向 `.cs`，正文可写「完整代码见 `XXX.cs`」。

---

### R3 · 模块划分

- 按**知识域**分 L2：当前为「基础语法」「面向对象」「实战案例」。
- 每个 L2 模块**必须有**速查卡（现文件名：`01-基础语法.md`、`02-面向对象.md`；可加 alias「核心速查卡」）。
- 一个知识点 = 一个 L3 文件；正课与练习可各一篇，同模块内并列。
- TLS_HX 内容按知识点就近合并进 Class02 对应笔记，不单独建平行模块。
- 新增或调整模块时：更新 README 子级简报与相关枢纽，不改写本规则的原则条文。

---

### R4 · 文件间链接类型

每篇 L3 至少建立以下边中的 **2 类**：

| 边类型 | 写法 | 用途 |
|--------|------|------|
| **序链** | 首尾 `[上一篇](./x.md) · [下一篇](./y.md)` | 同模块内学习顺序 |
| **索引边** | L2 速查卡 → L3 | 模块内复习入口 |
| **枢纽边** | L1 枢纽 → 跨模块 L3 | 跨域概念复习 |
| **前置边** | NOTE 中 `前置：[…](./…)` | 依赖关系 |
| **课件边** | frontmatter `对应课件` / 正文「完整代码见」 | 溯源，不进主链 |

**路径预算：** 任意 L3 到相关 L3，最短路径 ≤ **3 跳**（L3→L2 速查卡→L3，或 L3→L1 枢纽→L3）。超过 3 跳 → 补枢纽或调整模块，禁止加 L4 目录。

**链接策略：** 主链使用 **markdown 相对路径**（保证 GitHub 可渲染）；不强制 wikilink。Obsidian 图谱靠标签、入链与 Base。

---

### R5 · 枢纽规则

- L1 枢纽数量控制在 **4–8 个**；每个枢纽覆盖 **≥3 篇 L3**，且主题宜跨子域（同模块内用速查卡即可）。
- 枢纽页遵循 R0：`定义链`（本级）→ `核心篇目`（各 L3 一行简报 + 链接）→ `口诀` → `（可选）检查表链接`。
- **跨模块 / 跨子域复习走枢纽**；**按课序学习走模块序链**；二者不混用。
- 枢纽名单以 `Notes/枢纽/00_枢纽索引.md` 为现状真源。

---

### R6 · 文件命名（本仓库约定）

- L2 速查卡：`01-基础语法.md`、`02-面向对象.md`（保留现名，不强制改为 `00_核心速查卡.md`）
- L3 篇目：描述性中文名（如 `索引器.md`）；**不强制**模块内数字前缀
- 枢纽：`枢纽_概念名.md`，索引入口 `00_枢纽索引.md`
- 检查表：动词短语文件名（如 `跑通一节课.md`）
- **禁止**仅用编号作链接目标（如 `→ 01`）

---

### R7 · 信息密度

- **目标：** 有效链接数 / 文件数 ≥ 2；无孤立 L3（零出链且零入链的篇目）。
- **信息密度 = 高信噪比 + 可扫读结构**，不是把字塞进更少行。
- **单篇预算：L3 全文 ≤ 60 行**（含 frontmatter 与导航；图片/音频嵌入行不计入）。**60 行是上限不是目标**——禁止为凑行数合并并列知识点。超出 → 先砍案例全文与课堂流水账，再压踩坑，最后压 Python 对照；**不回退成挤排版**。
- **L1/L2 索引豁免**行数预算；仍须遵守「本级详述 + 子级简报」，不堆 L3 正文。
- **案例代码**只贴核心片段 **10–15 行**，注明「完整代码见 `XXX.cs`」。
- **宁表勿串**：并列参数 / 步骤 / 对比 → Markdown 表格或短 bullet。禁止一条 bullet 内用 ≥2 个中文分号「；」串联多个独立点。
- **提炼而非搬运**：课程原始知识点（概念/作用/特点/注意）逐条保留；口语展开压成一行。
- 踩坑一行一条：**症状 → 原因 → 修法**。
- Python 对照 ≤3 行，能一句话就一句话。
- L2 / L1 子级简报：`[**标题**](./篇目.md)：一句话描述`。
- 新增 L3 后必须同步：模块速查卡 +（若跨子域）对应枢纽 + README 模块简报（若新模块）。

## 二、Obsidian 优化规则

### O1 · Frontmatter

```yaml
---
tags:
  - 类型/知识点
  - 主题/类与对象
对应课件: TLS_HX/Lesson06_封装_索引器.cs
aliases:
  - indexer
---
```

| 规则 | 说明 |
|------|------|
| 类型标签 | 每篇恰好 **1 个** `类型/` |
| 主题标签 | 非索引笔记恰好 **1 个** `主题/`；索引（速查卡、枢纽、README、检查表）**不打主题标签** |
| 受控词表 | 新增 `类型/` 或 `主题/` 前先更新 `Notes/README.md` 词表与本文件 |
| 正文禁标签 | 不在正文散落 `#标签` |
| 对应课件 | 课例 `.cs` 相对仓库根路径（多个用列表） |
| aliases | 英文/别名，便于搜索 |

**类型词表：** `类型/知识点` · `类型/练习` · `类型/实战` · `类型/索引`

**主题词表：** `主题/语法基础` · `主题/流程控制` · `主题/数组` · `主题/方法` · `主题/类与对象` · `主题/类型系统` · `主题/继承多态` · `主题/泛型` · `主题/集合` · `主题/委托` · `主题/工具链`

---

### O2 · 标签与 Bases 分组

- **刻意只打 1 个主题标签**：Bases 对多值 tags 按「整个组合」分组。
- 维护 `Notes/课程索引.base`：按主题分组；过滤 `类型/索引`。
- README 嵌入 `![[课程索引.base]]`；手工模块简报 + Base 双轨。

---

### O3 · 链接写法

| 场景 | 正确 | 错误 |
|------|------|------|
| 速查卡索引 | `[**索引器**](./Class02-面向对象/索引器.md)：对象像数组访问…` | 只写「见索引器」无链接 |
| 前置边 | `前置：[成员属性](./成员属性.md)` | 裸 wikilink 作主链唯一形式 |
| 跨子域 | 经枢纽中转 | L3 直接堆 10+ 个跨域链接 |
| 图片/音频 | `![[文件名]]` 或相对路径 | 正文堆会过期的外链图床 |

- 链接目标必须是**真实存在的 `.md` 文件**。

---

### O4 · Callout 用法

| Callout | 用途 | 禁止内容 |
|---------|------|----------|
| `[!NOTE]` | 对应课件 + 一句话概括、前置/相关枢纽 | 元叙述（「浓缩索引」等） |
| `[!WARNING]` | 易错点：`症状 → 原因 → 修法` | 长段过程叙事 |
| `[!TIP]` | Python 对照、口诀、一行心路 | 完整课堂故事 |

callout 标记只用大写 NOTE/WARNING/TIP/IMPORTANT/CAUTION（GitHub 兼容子集），标记行不写自定义标题。

---

### O5 · 导航块

- **序链导航**：标题下（或文首 NOTE 后）一行  
  `[上一篇：标题](./上篇.md) · [下一篇：标题](./下篇.md)`
- 首篇只写「下一篇」；末篇只写「上一篇」。
- **主题关联 / 枢纽**放在 NOTE 的「相关」行或文末 `## 关联篇目`，不与序链混排成一团。

---

### O6 · 资源与 Attachments

- 音频/截图进 `Notes/Attachments/`，笔记中本地嵌入。
- 不入库：超大未压缩录屏、未授权商用整图。

---

### O7 · 检查表（行动层）

- 动手操作（跑通课例、写一篇笔记）→ `Notes/检查表/`，L1 入口链入。
- 检查表是**步骤清单**，不是知识推导；L3 可链到检查表，不把步骤塞进 L3 正文。

---

### O8 · Aliases

- 枢纽页、高频概念设 `aliases`（如 `List`、`delegate`）。
- L3 仅在标题易混淆时加 alias。

---

### O9 · 图谱与反向链接

- 依赖 **markdown 链接 + 受控标签** 驱动可发现性；打开 L3 时应能从速查卡/枢纽/前置入链。
- 若某 L3 入链为空 → 违反 R7，补索引边或枢纽边。

---

### O10 · 发布与校验清单

每篇 L3 完成后检查：

- [ ] frontmatter：1 类型 +（非索引）1 主题；`对应课件` 已填（若有课例）
- [ ] 链接可点开，无死链（markdown 相对路径）
- [ ] 速查卡已更新（若为新篇）
- [ ] 跨子域概念已更新枢纽
- [ ] Base 能自动收录（tags 符合词表）
- [ ] 无正文 `#标签`
- [ ] L3 ≤ 60 行（图/音频行可不计）；代码仅核心片段
- [ ] 有序链（同模块）或已说明为独立实战篇

## 三、整理工作流（课例 → Notes）

1. **读课例**：`Class01|Class02|TLS_HX|Example` 下对应 `.cs`。
2. **通读筛除**：剔除闲聊与重复；保留定义、语法、要点、易错点。
3. **归类分篇**：按知识域归入 L2；TLS_HX 就近并入 Class02。
4. **写作**：按 R7 与 O1–O5 写 L3；参照 `Notes/模板/知识点笔记.md`。
5. **联通**：跨子域进 `枢纽/`、步骤进 `检查表/`；同步速查卡 / 枢纽 / README 简报。
6. **校验**：跑 O10 清单。

## 五、规则优先级（冲突时）

1. **R0 本级详述 + 子级简报**
2. **R1 三层深度** — 禁止 L4
3. **R2 存储分工** — 源码不进主链
4. **O2 单主题标签**
5. **R4 路径 ≤3 跳** + markdown 主链
6. **O3 真实链接**
7. 其余规则按需执行
