# struct 结构体

> 对应 `Class02/Lesson17.cs`。结构体 = 自定义的**值类型**（和 class、enum 一样属于「类型定义」，不是集合）。

```csharp
struct Point                 // 写法和 class 几乎一样，关键字换成 struct
{
    public int X;
    public int Y;
    public Point(int x, int y) { X = x; Y = y; }   // 也可以有构造函数、方法
}

// 值类型的体现：赋值 = 复制一份，两者独立
Point p2 = p1;
p2.X = 100;
Console.WriteLine(p1.X);   // 3，p1 不受影响！（class 则传地址，会连带变 100）
```

## struct vs class（核心对比）

本质区别一条：**struct 是值类型，class 是引用类型**，其余差异都由此派生。

| | `struct` | `class` |
|---|---|---|
| 赋值 `b = a` | **复制一份**，两者独立 | 复制**地址**，指向同一对象 |
| 改 `b.X` 影响 `a.X`？ | ❌ 不影响 | ✅ 影响 |
| 继承类 | ❌ 不能 | ✅ 支持继承、多态 |
| 实现接口 | ✅ **可以** | ✅ 可以 |
| 可以为 `null` | ❌（除非 `Point?`） | ✅ |
| 适合 | 小而简单的数据（坐标、颜色、日期） | 复杂对象（默认选它） |

- **怎么选**：默认用 class；数据小、简单、纯粹表示一个值时才用 struct。
- 内置的 `int`、`double`、`bool`、`DateTime` 其实都是微软写好的 struct。

## 不能继承，但可以实现接口

```csharp
struct Point : IShow { public void Show() { } }   // ✅ 实现接口（可多个），写法同 class
// struct Point3D : Point { }                     // ❌ 不能继承，也不能被继承
```

- 没有 `virtual`/`override` 那套；实现接口是给 struct「附加能力」的唯一途径。
- 例如内置的 `int` 实现了 `IComparable` 等接口，所以能被排序、比较。

## struct vs List（不是一类东西）

- **struct** 是元素的**模板**（一个东西长什么样）；**List** 是装东西的**容器**。
- 配合使用：`List<Point> points = new List<Point>();`——列表装很多个结构体。
- 类比：struct/class 是「设计一种盘子」，List/Dictionary 是「放盘子的架子」。

## Python 对照

近似 `namedtuple` / `dataclass`，但它们仍是引用语义，没有 C# struct 的「赋值复制」行为。
