# 泛型 (Generics)

> 对应 `Class02/Lesson12.cs`。泛型 = 把「类型」也当成参数，写一份代码适配多种类型。

## 为什么需要泛型

不用泛型时，一个方法要支持多种类型，只能靠**重载**写很多份：

```csharp
public static void ShowSomething(int show)    { Console.WriteLine(show); }
public static void ShowSomething(string show) { Console.WriteLine(show); }
// ……每种类型都要抄一遍，重复
```

用泛型，一份就够，`T` 是「类型占位符」，调用时自动确定：

```csharp
public static void ShowSomething<T>(T show)
{
    Console.WriteLine($"ShowSomething: {show}");
}

ShowTool.ShowSomething(1);              // T = int
ShowTool.ShowSomething("Hello");        // T = string
ShowTool.ShowSomething(3.1415926);      // T = double
```

## 泛型约束 `where`

限制 `T` 必须满足某些条件，才能用它的成员：

```csharp
public static void ShowCarLogo<T>(T car) where T : Car
{
    Console.WriteLine($"Show Car Logo: {car.logo}");   // 约束了 T:Car，才能用 .logo
}
```

常见约束（`where T : 约束`，可多个）：

| 约束 | 含义 |
|------|------|
| `where T : struct` | T 必须是值类型 |
| `where T : class` | T 必须是引用类型 |
| `where T : new()` | T 必须有无参构造函数 |
| `where T : 基类名` | T 必须是该类或其子类 |
| `where T : 接口名` | T 必须实现该接口 |

## 泛型类

泛型也用于类，最常用的是集合 `List<T>`：

```csharp
List<int> nums = new List<int>();
List<string> names = new List<string>();
```

## 要点

- `<T>` 里 `T` 是类型参数（习惯用 `T`、`TKey`、`TValue`）。
- 好处：**减少重复** + **编译期类型安全**（比用 `object` 强，无需装箱/强转）。
- Python 对照：Python 动态类型天然“泛型”；类型提示 `list[int]`、`TypeVar` 概念上对应。
