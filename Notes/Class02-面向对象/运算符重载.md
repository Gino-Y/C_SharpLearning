# 运算符重载

> 对应 `TLS_HX/Lesson10_封装_运算符重载.cs`（唐老师核心课程）。运算符重载 = **让自定义类和结构体对象可以进行运算**——教会编译器 `p1 + p2` 这种式子该怎么算。

## 基本语法

使用关键字 `operator`：

```
public static 返回类型 operator 运算符(参数列表)
```

## 案例代码（TLS_HX/Lesson10）

```csharp
class Point
{
    public int x;
    public int y;

    // Point + Point
    public static Point operator +(Point p1, Point p2)
    {
        Point result = new Point();
        result.x = p1.x + p2.x;
        result.y = p1.y + p2.y;
        return result;
    }

    // Point + int
    public static Point operator +(Point p, int value)
    {
        Point result = new Point();
        result.x = p.x + value;
        result.y = p.y + value;
        return result;
    }

    // int + Point：运算符不会自动交换顺序，p + 10 和 10 + p 是两个签名，想都支持就都要写
    public static Point operator +(int value, Point p)
    {
        return p + value;   // 复用上面的重载，不用重抄一遍逻辑
    }
}

// 使用
Point p3 = p1 + p2;      // 编译器翻译成 Point.operator+(p1, p2)
Point p4 = p1 + 10;
Point p5 = 10 + p1;
```

## 特点

1. **一定是一个公共的静态方法**（`public static` 缺一不可，漏了直接报错）
2. **返回值写在 `operator` 前**
3. **逻辑处理自定义**（加法怎么算由你说了算）

## 注意

1. **条件运算符需要成对实现**：重载了 `==` 就必须重载 `!=`，`<` 配 `>`，`<=` 配 `>=`
2. **一个符号可以多个重载**（参数类型/顺序不同即可，如上面 `+` 写了三个版本）
3. **不能使用 `ref` 和 `out`** 修饰参数

## 踩过的坑（都来自 Lesson10 实操）

- **漏写 `static`** → `User-defined operator must be declared static and public`。为什么强制 static：执行 `p1 + p2` 时这个"+"不属于任何对象，编译器按 `Point.operator+(p1, p2)` 静态调用，操作数全靠参数传入。
- **`new Point(a, b)` 报参数不符** → 类里没写带参构造函数。补带参构造后记得无参的也要手动写（写了带参，编译器就不送无参了——`Player` 练习和这里连踩两次）。
- **参数和局部变量重名** → `Conflicting variable 'p' is defined below`。参数叫 `p`，方法体里又 `Point p = new Point()`，同作用域两个 `p` 编译不过；而且就算能编过，`p.x + value` 读的也是空对象而不是参数。结果对象换名叫 `result` 就清晰了。

## Python 对照

```python
class Point:
    def __add__(self, other):    # p + other
        ...
    def __radd__(self, other):   # other + p（右加），对应 C# 的 (int, Point) 重载
        ...
```

C# 用 `operator +` 静态方法，Python 用 `__add__` 魔术方法；C# 的"顺序不自动交换"在 Python 里对应 `__add__` / `__radd__` 之分，思想完全一致。
