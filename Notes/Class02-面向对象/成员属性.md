# 成员属性（get / set）

> 对应 `TLS_HX/Lesson05_成员属性.cs`（唐老师核心课程）。属性 = **给私有字段包一层可控的门**：外部看着像变量，实际走 get/set。前置：[类与对象](./类与对象.md)（那里的自动属性是本篇的简写形态）。

## 基本概念

1. **用于保护成员变量**（字段设 `private`，外部只能通过属性进出）
2. **为获取和赋值添加逻辑处理**（校验、加密等）
3. **解决 3P 的局限性**：`public` 内外访问 / `private` 内部访问 / `protected` 内部和子类访问——3P 只能"全开或全关"，属性能做到**只能获取不能修改**，或**只能修改不能获取**

## 基本语法

```
访问修饰符 属性类型 属性名 { get{} set{} }
```

核心片段（完整代码见 `TLS_HX/Lesson05_成员属性.cs`，含 Money 加密解密例子）：

```csharp
private string name;              // 字段：私有，藏在类内部
public string Name                // 属性：命名首字母大写
{
    get { return name; }          // 获取前可加逻辑
    set                           // value 关键字 = 外部赋过来的值
    {
        if (value.Length > 10) return;   // 赋值前可加校验
        name = value;
    }
}
// 使用：person.Name = "张三" 触发 set；读 person.Name 触发 get
```

## get 和 set 前可以加访问修饰符

```csharp
public int Age { get; private set; }   // 外部能查不能改
```

1. 默认不加，使用属性声明时的访问权限
2. 加的访问修饰符要**低于**属性的访问权限
3. 不能让 get 和 set 的访问权限**都**低于属性的权限

- 踩坑：`private set` 后类外赋值 → 编译报错 CS0272 → 保护是编译期的，不是运行时报警。

## get 和 set 可以只有一个

- 只有一个时没必要在前面加访问修饰符；一般只会出现**只有 get 没有 set**（只读）
- `get => sex;` 是 `get { return sex; }` 的表达式简写

## 与自动属性的关系

- `{ get; set; }` 自动属性 = 编译器暗中生成私有字段和直进直出的 get/set
- 选择：get/set 不需要逻辑 → 自动属性；需要校验/加密 → 本篇完整写法

## Python 对照

`@property` ≈ get，`@name.setter` ≈ set；C# 的 `value` 关键字 ≈ setter 的参数。
