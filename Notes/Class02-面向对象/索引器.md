# 索引器（this[...]）

> 对应 `TLS_HX/Lesson06_封装_索引器.cs`（唐老师核心课程）。索引器 = **让对象可以像数组一样通过索引访问其中元素**，程序更直观、更容易编写。前置：[成员属性](./成员属性.md)（get/set 写法和规则相同）。

## 基本语法

```
访问修饰符 返回值 this[参数类型 参数名, 参数类型 参数名.....]
{
    get{}
    set{}      // 内部写法和规则与成员属性相同，value 代表传入的值
}
```

核心片段（完整代码见 `TLS_HX/Lesson06_封装_索引器.cs`）：

```csharp
class Person
{
    private Person[] friends;
    private int[,] array = new int[10, 10];

    public Person this[int index]              // 索引器
    {
        get
        {
            // 知识点四：索引器中可以写逻辑，规则自己定
            if (friends == null || friends.Length - 1 < index) return null;
            return friends[index];
        }
        set { /* null 时新建 / 越界顶掉最后一个 / 正常写入 */ }
    }

    public int this[int i, int j]              // 重载：参数数量不同
    {
        get { return array[i, j]; }
        set { array[i, j] = value; }
    }

    public string this[string str] { get { /* 按 "name"/"age" 返回字段 */ } }   // 重载：参数类型不同
}

// 使用：像数组一样用中括号
p[0] = new Person();      // 走 this[int] 的 set
p[0, 0] = 10;             // 走 this[int, int] 的 set
Console.WriteLine(p["name"]);   // 走 this[string] 的 get
```

## 要点

- **可以写逻辑**：get/set 里能做判空、越界处理、按需返回——规则自己定。
- **可以重载**：函数名相同（都是 `this`），参数类型、数量、顺序不同即可。
- **适用场景**：类中有数组变量时，方便访问并统一做逻辑处理。
- **结构体也支持索引器**。

## 踩过的坑（Lesson06 实操）

- `p[0,0]=10` 抛 NullReferenceException → 字段 `int[,] array` 只声明没 `new`（引用类型默认 null）→ 声明时初始化。
- 越界"顶掉最后一个"分支执行完仍继续越界赋值 → 分支穿透 → 正常写入放进 `else`。

## Python 对照

≈ `__getitem__` / `__setitem__` 魔术方法（`p[0]` 同款体验），C# 版本多了重载和编译期类型检查。
