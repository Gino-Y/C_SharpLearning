# 拓展方法（扩展方法）

> 对应 `TLS_HX/Lesson09_封装_拓展方法.cs`（唐老师核心课程）。拓展方法 = **给现有类型"外挂"新方法**，不改源码、不用继承，调用起来就像类型自带的一样。

**前置知识**：[static 静态成员与静态类](./static静态成员.md)。知识链：

```
静态成员 → 静态方法（工具函数） → 静态类（工具类） → 拓展方法（本篇）
```

拓展方法**就是静态类里的静态方法**，只多一个记号：第一个参数加 `this`。加了之后调用方式升级——`MathTool.GetABS(x)` 变成 `x.GetABS()`。

## 概念

为现有**非静态**的类型（哪怕是别人封装好的、你没有源码的类型，如 `string`、`int`）添加新方法。

## 基本语法

```
访问修饰符 static 返回值 函数名(this 拓展类名 参数名, 参数类型 参数名, 参数类型 参数名....)
```

对照实例逐段拆解：

```csharp
public   static  void   SpeakStringInfo (this string source, string str2, string str3)
// 访问修饰符 static 返回值  函数名        this 拓展类名 参数名   后续正常参数...
```

## 案例代码（TLS_HX/Lesson09）

```csharp
// 拓展方法必须写在静态类中
static class Tools
{
    // 给 int 拓展：成员方法需要实例化对象，拓展方法不需要改类型源码
    public static void SpeakValue(this int value)
    {
        Console.WriteLine($"The value is {value}");
    }

    // 给 string 拓展：可以带额外参数（从第二个参数开始是正常参数）
    public static void SpeakStringInfo(this string source, string str2, string str3)
    {
        Console.WriteLine($"调用方法的对象： {source}");
        Console.WriteLine($"传的参数： {str2}");
        Console.WriteLine($"传的参数： {str3}");
    }

    // 给自定义类 Test 拓展：与成员方法 Func2 重名，见下方「要点」
    public static void Func2(this Test source)
    {
        Console.WriteLine($"789");
    }
}

public class Test
{
    public void Func1() { Console.WriteLine($"123"); }
    public void Func2() { Console.WriteLine($"456"); }
}

// 调用：像类型自带的方法一样用「对象.方法()」
int value = 10;
value.SpeakValue();                        // The value is 10
"Hello".SpeakStringInfo("A", "B");         // 调用时不传第一个参数，"Hello" 就是第一个参数

Test test = new Test();
test.Func2();          // 456 —— 与成员方法重名，成员方法优先
Tools.Func2(test);     // 789 —— 用静态原始调法仍能调到拓展方法
```

## 作用

1. 提升程序拓展性
2. 不需要在对象中重新写方法
3. 不需要继承来添加方法
4. 为别人封装的类型写额外的方法（最常用：给 `string`、`int` 或第三方库的类加方法）

## 特点（四个硬性规则）

1. **一定写在静态类中**（`static class`）
2. **一定是静态函数**（`public static`）
3. **第一个参数为拓展目标**（想给谁加方法，第一个参数就是谁的类型）
4. **第一个参数用 `this` 修饰**（这是拓展方法的标志）

## 注意

- **可以有返回值和 n 个参数，根据需求而定**——返回值不限 `void`，`this` 参数后面可以接任意个正常参数（如 `SpeakStringInfo` 带了两个）。

## 要点

- 调用时第一个参数**不用传**，点号前面的对象自动作为第一个参数传入——`value.SpeakValue()` 等价于 `Tools.SpeakValue(value)`。
- 本质是编译器的语法糖：编译后还是普通静态方法调用。
- 如果拓展方法和类型自带方法同名，**成员方法优先，拓展方法被压制（不是覆盖）**——`test.Func2()` 走成员方法输出 456；改用静态原始调法 `Tools.Func2(test)` 仍可调到，输出 789。
- 拓展目标不限内置类型：`string`、`int`、自定义类（如 `Test`）都可以。
- 学过的 LINQ 风格方法（如 `List` 的很多操作）底层大量使用拓展方法。
- Python 对照：Python 可以直接给类动态加方法（monkey patch），C# 类型是封闭的，拓展方法是官方提供的"安全外挂"方式。

## 顺带：#region 折叠注释

截图里的 `#region 知识点一 ... #endregion` 不是拓展方法的语法，是**代码折叠标记**：IDE 里可以把这一段代码折叠成一行，方便组织长文件，编译时完全忽略。
