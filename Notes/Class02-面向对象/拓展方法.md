# 拓展方法（扩展方法）

> 对应 `TLS_HX/Lesson09_封装_拓展方法.cs`（唐老师核心课程）。拓展方法 = **给现有类型"外挂"新方法**，不改源码、不用继承，调用起来像类型自带的。

**前置**：[static 静态成员与静态类](./static静态成员.md)。知识链：静态成员 → 静态方法 → 静态类 → 拓展方法（本篇）。本质：**静态类里的静态方法 + 第一个参数加 `this`**，调用从 `Tools.GetABS(x)` 升级成 `x.GetABS()`。

## 概念

为现有**非静态**的类型（哪怕是别人封装好的，如 `string`、`int`）添加新方法。

## 基本语法

```
访问修饰符 static 返回值 函数名(this 拓展类名 参数名, 参数类型 参数名....)
```

核心片段（完整代码见 `TLS_HX/Lesson09_封装_拓展方法.cs`）：

```csharp
static class Tools                              // 必须是静态类
{
    public static void SpeakValue(this int value)          // 给 int 拓展
    {
        Console.WriteLine($"The value is {value}");
    }
    public static void Func2(this Test source)             // 给自定义类拓展
    {
        Console.WriteLine($"789");
    }
}

10.SpeakValue();       // 调用时不传第一个参数，10 本身就是 value
test.Func2();          // 456 —— 与成员方法重名时成员方法优先
Tools.Func2(test);     // 789 —— 静态原始调法仍能调到拓展方法
```

## 作用

1. 提升程序拓展性
2. 不需要在对象中重新写方法
3. 不需要继承来添加方法
4. 为别人封装的类型写额外的方法（最常用）

## 特点（四个硬性规则）

1. **一定写在静态类中**（`static class`）
2. **一定是静态函数**（`public static`）
3. **第一个参数为拓展目标**
4. **第一个参数用 `this` 修饰**

## 注意

- **可以有返回值和 n 个参数，根据需求而定**——`this` 参数后面可接任意个正常参数。

## 要点

- 本质是语法糖：`x.GetABS()` 编译后就是 `Tools.GetABS(x)`。
- 与自带方法同名 → **成员方法优先，拓展方法被压制（不是覆盖）**，静态原始调法仍可调到。
- 拓展目标不限内置类型：`string`、`int`、自定义类都行。
- LINQ 的 `Where`/`Select`/`Max` 等查询方法 = 官方给 `IEnumerable<T>` 写的一大批拓展方法。
- 顺带：`#region 名字 ... #endregion` 是 IDE 代码折叠标记，编译时忽略，不是拓展方法语法。
- Python 对照：≈ monkey patch（动态加方法），但 C# 是编译期检查的"安全外挂"。
