# 委托练习 3：委托当参数传递（自制筛选器）

> [!NOTE]
> 前两题委托是「字段 + 注册」，这次练委托的另一大用法：**把方法当参数传给别的方法**。
> 前置：[delegate 委托](./委托.md) · [委托练习 2](./委托-练习2.md) · 相关：[泛型](./泛型.md)（LINQ 的 `Func<int,bool>`）

## 场景

你有一个成绩列表，想按不同条件筛选：及格的、优秀的、不及格的……筛选的「遍历骨架」都一样，只有「判断条件」不同——把条件作为委托传进去，一个方法搞定所有筛选。

## 要求

### 1. 声明「判断型」委托

```csharp
public delegate bool ScoreFilter(int score);   // 收一个 int，返回 bool（是否保留）
```

> 考点一：委托可以**有返回值**。

### 2. 筛选方法（静态工具方法即可）

```csharp
static List<int> Filter(List<int> scores, ScoreFilter condition)
```

- 遍历 `scores`，对每个分数调用 `condition(score)`，返回 `true` 的收进结果列表
- 注意：这里委托是**方法参数**，不是字段，也不用 `+=`

> 考点二：委托类型和 `int`、`string` 一样，可以当参数类型用——这就是「把行为传进方法」。

### 3. 条件方法（写两个普通静态方法）

- `IsPass(int score)` → `score >= 60`
- `IsExcellent(int score)` → `score >= 90`

### 4. `Run()` 方法

1. 成绩列表：`{ 55, 78, 92, 45, 88, 96, 61 }`
2. 用 `Filter(scores, IsPass)` 筛及格 → 打印结果
3. 用 `Filter(scores, IsExcellent)` 筛优秀 → 打印结果
4. **挑战**：不写新方法，用 **Lambda 表达式**直接筛「不及格」：

```csharp
List<int> failed = Filter(scores, score => score < 60);
```

> 考点三：Lambda `x => 条件` 就是「现场写一个匿名方法塞给委托」，不用专门定义方法。

## 预期输出

```
及格: 78 92 88 96 61
优秀: 92 96
不及格: 55 45
```

## 提示（卡住再看）

- `Filter` 内部就是普通 `foreach` + `if (condition(score))`。
- 传条件时**只写方法名**不加括号：`Filter(scores, IsPass)`——加括号变成「调用」，不加才是「把方法本身递过去」。
- 这题练完你就理解了 LINQ 的 `Where(x => ...)` 底层是怎么回事（`Func<int, bool>`）。
