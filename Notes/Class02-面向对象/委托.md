# delegate 委托

> 对应 `Class02/Lesson19a.cs`（非委托版）与 `Lesson19b.cs`（委托版）。委托 = **能装方法的变量类型**——变量装数据，委托装「方法」，调用委托就等于调用它装的所有方法。

## 引子：Lesson19a 的写法（没用委托）

校长要通知所有教室下课，控制器**拿着教室列表，挨个点名调用**：

```csharp
class ClassController
{
    private List<ClassRoom> classRooms;   // 必须持有所有教室

    public void ClassOver()
    {
        foreach (ClassRoom classRoom in classRooms)
        {
            classRoom.ClassOver();        // 逐个调用
        }
    }
}
```

能用，但控制器和教室**耦合**：它必须知道「教室列表」这个具体类型，想再通知食堂、宿舍就得再加 List、再改 foreach。

## 委托写法（Lesson19b）：把「要做的事」注册进来

```csharp
class ClassControllerB
{
    // 1. 声明委托类型：规定能装什么样的方法（无参、无返回值）
    public delegate void ClassOverHandler();

    // 2. 委托变量：装一堆「下课时要执行的方法」（? 表示可能没人注册）
    public ClassOverHandler? OnClassOver;

    public void ClassOver()
    {
        OnClassOver?.Invoke();   // 3. 调用委托 = 广播，装的方法全部执行
    }
    // 注意：这里没有 List<ClassRoom> 字段，控制器不认识教室
}

// 使用：把每间教室的方法「+=」注册到委托上
ClassControllerB controller = new ClassControllerB();
for (int i = 101; i <= 111; i++)
{
    ClassRoom room = new ClassRoom(i.ToString());
    controller.OnClassOver += room.ClassOver;   // 注册（多播）
}
controller.ClassOver();   // 一声令下，11 间教室全部执行
```

两版行为完全相同（101~111 都下课），差别在结构：

| | Lesson19a 非委托 | Lesson19b 委托 |
|---|---|---|
| 控制器字段 | `List<ClassRoom>` | 只有委托变量 |
| 控制器认识教室吗 | ✅ 必须 | ❌ 不认识 |
| 建立联系的方式 | 构造时传列表 | 教室自己 `+=` 注册 |

- 控制器**不再需要知道教室是谁**，谁想被通知谁自己 `+=` 注册——食堂、宿舍也能注册，控制器代码一行不改。
- `+=` 注册、`-=` 取消注册；一个委托可装多个方法，称为**多播委托**。

## 要点

- 声明：`delegate 返回类型 名字(参数);` —— 定义「能装什么签名的方法」。
- 方法签名必须和委托**匹配**（参数、返回类型一致）才能装进去。
- 调用前最好判空：`OnClassOver?.Invoke();`（没人注册时直接调用会抛 NullReferenceException）。
- 系统内置了现成委托，通常不用自己声明：
  - `Action` / `Action<T>`：无返回值（`Action` ≈ 上面的 ClassOverHandler）
  - `Func<T, TResult>`：有返回值
- 委托是**事件 (event)** 的基础，下一步学 event 就是给委托加安全限制。

## 类比与对照

- 类比：委托像**微信群**——教室扫码进群（`+=`），校长在群里喊一嗓子（调用委托），所有群成员都收到；校长不用挨个存好友。
- Python 对照：Python 函数本身就是对象，可以 `handlers.append(room.class_over)` 再循环调用——C# 的委托就是把这件事**类型化**了（编译期检查签名）。
