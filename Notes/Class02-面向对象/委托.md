# delegate 委托

> [!NOTE]
> 对应 `Class02/Lesson19a.cs`（非委托版）与 `Lesson19b.cs`（委托版）。委托 = **能装方法的变量类型**——变量装数据，委托装「方法」，调用委托 = 调用它装的所有方法。
> 前置：[方法与方法重载](../Class01-基础语法/方法与方法重载.md)（委托装的就是方法）· [类与对象](./类与对象.md)

## 引子：非委托版的耦合问题（Lesson19a）

校长通知下课：控制器持有 `List<ClassRoom>`，foreach 挨个调 `classRoom.ClassOver()`。能用，但控制器和教室**耦合**——想再通知食堂、宿舍就得再加 List、再改 foreach。

## 委托写法（Lesson19b）：把「要做的事」注册进来

```csharp
class ClassControllerB
{
    public delegate void ClassOverHandler();   // 1. 声明委托类型：规定能装什么签名的方法
    public ClassOverHandler? OnClassOver;      // 2. 委托变量（? 表示可能没人注册）

    public void ClassOver()
    {
        OnClassOver?.Invoke();                 // 3. 调用委托 = 广播，装的方法全部执行
    }
    // 没有 List<ClassRoom> 字段，控制器不认识教室
}

// 使用：教室的方法「+=」注册进委托
controller.OnClassOver += room.ClassOver;   // 注册（多播）
controller.ClassOver();                     // 一声令下，注册过的全部执行
```

两版行为相同，差别在结构：

| | 非委托版 | 委托版 |
|---|---|---|
| 控制器字段 | `List<ClassRoom>` | 只有委托变量 |
| 控制器认识教室吗 | ✅ 必须 | ❌ 不认识 |
| 建立联系的方式 | 构造时传列表 | 教室自己 `+=` 注册 |

- 控制器**不需要知道教室是谁**，谁想被通知谁 `+=`——食堂、宿舍也能注册，控制器一行不改。
- `+=` 注册、`-=` 取消注册；一个委托装多个方法 = **多播委托**。

## 要点

- 声明：`delegate 返回类型 名字(参数);`——定义「能装什么签名的方法」。
- 方法签名必须和委托**匹配**（参数、返回类型一致）才能装进去。
- 调用前判空：`OnClassOver?.Invoke();`（没人注册时直接调用抛 NullReferenceException）。
- 内置委托通常不用自己声明：`Action`/`Action<T>` 无返回值；`Func<T, TResult>` 有返回值。
- 委托是**事件 (event)** 的基础，event = 给委托加安全限制。

> [!TIP]
> - 类比：委托像**微信群**——教室扫码进群（`+=`），校长群里喊一嗓子（Invoke），全员收到。
> - Python 对照：函数本身就是对象，`handlers.append(fn)` 再循环调用；C# 委托把这事**类型化**了（编译期查签名）。
