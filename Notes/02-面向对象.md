## Lesson01：类与对象

```csharp
class Student
{
    // 属性（自动属性 get/set）
    public string Name { get; set; }
    public int Age { get; set; }
    public string Gender { get; set; }
}

// 使用：
Student student = new Student();   // new 一个对象（实例）
student.Name = "张三";              // 用「对象.成员」赋值
student.Age = 18;
Console.WriteLine(student.Name);
```

要点：

- **类 (class)** = 模板/图纸；**对象 (object)** = 用 `new` 造出的具体实例。
- 每个对象的属性**互相独立**：`student` 和 `student2` 的 `Name` 互不影响。
- `public string Name { get; set; }` 是**自动属性**：编译器自动生成读(get)/写(set)。
- 类里放两类东西：**数据**（字段/属性）和 **行为**（方法）。

## Lesson04：构造函数

```csharp
class Car
{
    public string logo;

    public Car(string lg)   // 带参构造函数：new 时初始化
    {
        logo = lg;
    }

    public Car() { }        // 无参构造函数（重载）

    ~Car() { }              // 析构函数/终结器（极少用，可忽略）
}

// 使用：
Car car = new Car("Toyota");   // 调用带参构造，logo = "Toyota"
```

要点：

- 构造函数：**方法名 = 类名**、**没有返回类型**，`new` 对象时自动调用，用来初始化。
- **可以重载**：多个构造函数，参数不同即可（如 `Car(string)` 和 `Car()`）。
- **默认构造函数的陷阱**：
  - 一个构造函数都不写 → 编译器免费送一个无参构造。
  - 只要手写了任意一个构造函数 → 免费的无参版**消失**；还想用 `new Car()` 就得手动补 `Car()`。
- `~Car()` 是**析构函数**（对象被 GC 回收时调用），和构造函数相反，现代 C# 几乎用不到。

## Lesson05：static 静态成员

```csharp
public class Dog : Animal
{
    public int Age { get; set; }        // 实例成员：每个对象各一份
    public static int num { get; set; } // 静态成员：全类共享一份
}

class NumTool
{
    public static int Add(int a, int b) => a + b;   // 静态方法（工具）
}

// 使用：
Dog dog1 = new Dog();
dog1.Age = 3;        // 实例成员用「对象」访问
Dog.num++;           // 静态成员用「类名」访问
int c = NumTool.Add(3, 5);   // 静态方法不用 new
```

要点：

- **`static`（静态）= 属于类本身**；不带 static（实例）= 属于每个对象。
- 访问方式：
  - 静态成员 → **类名.成员**（`Dog.num`）
  - 实例成员 → **对象.成员**（`dog1.Age`）
- 静态字段**全类共享一份**、贯穿整个程序生命周期（常用作计数器）。
- 静态方法适合做**工具函数**（如 `Math.Max`、`Console.WriteLine`），不依赖具体对象。
- 陷阱：**静态方法里不能直接访问实例成员**（因为没有具体对象）。
- `Main` / `Run` 必须是 static，因为程序启动时还没有任何对象。

## Lesson06：值类型 vs 引用类型（栈与堆）

```csharp
// 值类型传参：复制「值」，方法内修改不影响外面
static void Exchange(int a, int b)
{
    int temp = a; a = b; b = temp;
}
int a = 10, b = 20;
Exchange(a, b);
// 结果：a=10, b=20（没变）

// 引用类型传参：复制「地址」，指向同一对象
static void Exchange(ZangAo a, ZangAo b)
{
    int temp = a.Age; a.Age = b.Age; b.Age = temp;
}
ZangAo x = new ZangAo { Age = 5 };
ZangAo y = new ZangAo { Age = 10 };
Exchange(x, y);
// 结果：x.Age=10, y.Age=5（真的交换了）
```

内存模型：

- **栈 (Stack)**：存局部变量。值类型直接存值；引用类型存的是「指向堆的地址」。
- **堆 (Heap)**：存 `new` 出来的对象，由 GC 管理。

| | 值类型（int、bool、struct） | 引用类型（class、数组、string） |
|---|---|---|
| 变量里存什么 | 值本身 | 对象的地址（引用） |
| 传参复制的 | 值的副本 | 地址的副本 |
| 方法内改动影响外面？ | ❌ 不影响 | ✅ 改「内容」会影响 |

关键陷阱：

- 改 `a.Age`（顺着地址改堆里的**内容**）→ 外面**能**看到。
- 改 `a` 本身（`a = b`，只换**地址副本**的指向）→ 外面**看不到**。

## Lesson07 / 07b：继承、多态、抽象类

### 继承

```csharp
public class Cat : Animal   // Cat 继承 Animal，冒号读作「继承自」
{
    // 自动拥有 Animal 的 name、Eat()，可重写 virtual/abstract 方法
}
```

- C# **只能单继承**（冒号后只能有一个父类）；多来源用「接口 interface」。
- 子类自动获得父类的 `public`/`protected` 成员。

### virtual / override / abstract

```csharp
public abstract class Animal      // 抽象类：不能被 new
{
    public string name;

    public void Eat()             // 普通方法：不能被 override
    {
        Console.WriteLine($"{name} is eating.");
    }

    public abstract void Sleep(); // 抽象方法：无实现，子类【必须】重写

    public virtual void Walk()    // 虚方法：有默认实现，子类【可以】重写
    {
        Console.WriteLine($"{name} is walking.");
    }
}

public class Cat : Animal
{
    public override void Sleep()  // 必须实现抽象方法
    {
        Console.WriteLine($"{name} sleep: huhuhu~");
    }
    public override void Walk()   // 可选：重写虚方法
    {
        Console.WriteLine($"{name} is walking~~~.");
    }
}
```

| 关键字        | 位置                 | 有方法体    | 含义                     |
| ---------- | ------------------ | ------- | ---------------------- |
| `virtual`  | 父类方法               | ✅ 有默认实现 | 允许子类重写，不重写就用默认         |
| `abstract` | 父类方法（类也须 abstract） | ❌ 无     | 强制子类重写                 |
| `override` | 子类方法               | ✅       | 重写父类的 virtual/abstract |

- 要用 `override` 重写，父类方法**必须**是 `virtual` 或 `abstract`；普通方法不能被重写。
- **抽象类**（`abstract class`）不能直接 `new`，只能被继承。

### 多态 / 里氏转换原则

```csharp
Animal mimi = new Cat();   // 父类变量指向子类对象（里氏转换）
mimi.name = "Mimi";
mimi.Sleep();              // 运行时调用 Cat 的 Sleep（动态分派）
mimi.Walk();               // 调用 Cat 的 Walk
```

- **里氏转换原则**：子类对象可以赋给父类类型的变量（`Animal a = new Cat();`）。
- **多态**：通过父类变量调用 virtual/abstract 方法时，运行时会执行**对象真实类型**对应的版本。
- 好处：可以用统一的父类类型处理不同子类，无需逐个 `if` 判断类型。

```csharp
// Lesson07b：用父类类型参数接收任意子类
static void BeginCar(CarA car) { car.Begin(); }
BeginCar(new Audi());   // 执行 Audi 的 Begin
BeginCar(new Auto());   // 执行 Auto 的 Begin
```

## 接口 interface

> 对应 `Class02/IHotChair.cs` 和 `Lesson07b.cs` 里 `Audi` 的实现。

```csharp
// 定义接口：只声明「能做什么」，不写实现
public interface IHotChair
{
    void HotChair();   // 无方法体、无访问修饰符（默认 public）
}

// 类实现接口：基类在前，接口在后，用逗号隔开
class Audi : CarA, IHotChair
{
    // 实现接口方法：必须 public，方法名/签名要一致
    public void HotChair()
    {
        Console.WriteLine($"{nameof(Audi)}有座椅加热");
    }
    // ... 还有从 CarA 继承/重写的 Begin、ChangeGear
}

// 使用：
Audi audi = new Audi();
audi.HotChair();       // 座椅加热（Audi 独有的附加功能）
```

要点：

- **接口 (interface)** 只规定「必须有哪些方法/属性」（一份契约），**不含实现**（成员没有方法体）。
- 命名习惯：接口名以大写 **`I`** 开头（`IHotChair`）。
- 接口成员**默认 `public`**，声明时不写修饰符；实现类里对应方法**必须写 `public`**。
- **类可以实现多个接口**（弥补 C# 只能单继承的限制）：
  - `class Audi : CarA, IHotChair` —— 一个基类 `CarA` + 一个接口 `IHotChair`。
  - 语法上**基类必须写在最前面**，接口跟在后面，可以有多个：`: 基类, 接口1, 接口2`。
- 接口的意义：给不同的类**附加统一的能力**。比如只有部分车有「座椅加热」，就让它们都实现 `IHotChair`，其它车不用管。
- 接口也是**多态**的一种：可以用接口类型的变量指向任意实现了它的对象（`IHotChair x = new Audi();`），方便里氏转换。

### 接口 vs 抽象类

| | 接口 interface | 抽象类 abstract class |
|---|---|---|
| 能否有实现 | 传统上不能（只声明） | 可以有普通方法的实现 |
| 一个类能有几个 | **多个**接口 | 只能继承**一个**抽象类 |
| 成员默认可见性 | public | 需显式写 |
| 用途 | 附加「能做什么」的能力 | 表达「是什么」的父子关系 + 共享代码 |
| 关键字（子类） | 直接实现（写 `public`） | `override` 重写抽象方法 |

> 小工具：`nameof(Audi)` 返回字符串 `"Audi"`（类名），比手写字符串更安全——改类名时它会跟着变，不会写错。

课程代码中的类形成了一条继承链（跨多个文件，同属 `CSharpLearning.Class02`）：

```

```
Car  (Lesson04)

## 与 Python 快速对照

| 概念    | Python                         | C#                                     |
| ----- | ------------------------------ | -------------------------------------- |
| 构造函数  | `__init__`                     | `public 类名(...)`                       |
| 实例/静态 | `self.x` / 类变量、`@staticmethod` | 实例成员 / `static`                        |
| 继承    | `class Dog(Animal)`            | `class Dog : Animal`                   |
| 多继承   | 支持                             | 不支持（单继承 + 接口）                          |
| 方法重写  | 默认可重写（无需关键字）                   | 父类须 `virtual`/`abstract`，子类 `override` |
| 抽象方法  | `@abstractmethod`              | `abstract`（在 `abstract class` 中）       |
| 接口    | `Protocol` / 抽象基类              | `interface`（一个类可实现多个）                  |
