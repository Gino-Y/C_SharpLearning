# C# 学习笔记

根据本项目 `Class01/`、`Class02/`（小M老师课程）、`TLS_HX/`（唐老师C#核心课程）、`Example/` 下的课程代码整理。TLS_HX 的内容按知识点就近合并进对应笔记。

## 笔记组织方式

- **一个知识点 = 一个独立文件**，放在对应模块的子文件夹里。
- 模块索引文件（本文件、`01-基础语法.md`、`02-面向对象.md`）只做**标题、概述、引用**，不堆内容。

## 单篇笔记模板（防膨胀标准）

- **单篇 ≤60 行**；超线先砍案例代码，再压踩坑，最后压 Python 对照。
- 案例代码只贴**核心片段 10–15 行**，注明「完整代码见 `XXX.cs`」，不贴全文。
- 踩坑一行一条：**症状 → 原因 → 修法**，不写过程叙述。
- Python 对照 ≤3 行，能一句话就一句话。
- 课程原始知识点（概念/作用/特点/注意）逐条保留，不删减。
- 知识链/前置链接保留（每条 1 行）。

## 标签规范

标签写在每篇笔记的 frontmatter `tags:` 里，两个维度，**受控词表**——新增标签必须先改本节。

**规则**：每篇恰好 1 个 `类型/` 标签；知识点/练习/实战笔记再加恰好 1 个 `主题/` 标签（索引文件不打主题标签）。不在正文里散落 `#标签`。

- **类型维度（4 个）**：`类型/知识点`、`类型/练习`、`类型/实战`、`类型/索引`
- **主题维度（11 个）**：`主题/语法基础`、`主题/流程控制`、`主题/数组`、`主题/方法`、`主题/类与对象`、`主题/类型系统`、`主题/继承多态`、`主题/泛型`、`主题/集合`、`主题/委托`、`主题/工具链`

> 只打 1 个主题标签（不打多个）是刻意的：Obsidian Bases 对多值列表按"整个组合"分组，多主题标签会产生无意义的组合分组。`课程索引.base` 依赖这条规则做按主题分组。
